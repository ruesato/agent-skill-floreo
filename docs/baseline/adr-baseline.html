ADR-042: Adopt Dolt for Beads Storage

Switch beads issue tracker from SQLite to Dolt for distributed sync

Status

Accepted — 2026-05-24

Context

The beads issue tracker stores data in a local SQLite file. Multi-device workflows require manual export/import. Git-native syncing via Dolt would eliminate this friction and allow collaborative issue tracking without a central server.

Decision

Replace SQLite with Dolt (a MySQL-compatible versioned database). Issues are committed as Dolt transactions. Sync happens via dolt push/pull against a shared remote.

Alternatives Considered

OptionProsConsRejected Because
SQLite + git-tracked fileSimpleMerge conflicts on binary fileBinary format not diffable
Central PostgresEasy syncRequires serverAdds infra dependency
DoltGit semantics, diffableLearning curveAccepted

Consequences

  • Sync requires dolt CLI installed
  • Row-level diffs instead of file diffs
  • Branching model unlocks multi-agent workflows
Note: Dolt is API-compatible with MySQL — existing SQL queries required no changes.
