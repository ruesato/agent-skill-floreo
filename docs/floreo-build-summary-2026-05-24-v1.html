What Floreo Is

Floreo is a Claude Code plugin skill that turns agent-created content into polished, self-contained HTML reading artifacts. Where agents default to Markdown, floreo produces structured HTML with a built-in design system: typography, color tokens, components (tables, callouts, timelines, code blocks, stat blocks), dark mode, and responsive layout. Every document is a single .html file — no external dependencies, opens offline, under 100KB.

The core thesis: the same information, arranged so its meaning can finally be seen. Markdown is for source control; floreo documents are for reading.

Floreo is NOT for: .md files a project or tool expects (README.md, CLAUDE.md, SKILL.md, steering files), skill definitions, or content a downstream agent will parse as text.

Design Identity

Floreo documents share a consistent visual register: editorial and serious, not decorative. System fonts — Georgia for headings, system-ui for body, Courier New for code/metadata/eyebrows — with no external font load. Warm stone neutrals (#fafaf9 background, #1c1917 text, #57534e muted text). Accent color varies by document type to create visual identity at a glance. Dark mode via prefers-color-scheme. Max width 860px for comfortable reading.

The design deliberately avoids gradient text, glassmorphism, hero-metric templates, identical card grids, and side-stripe borders on callouts. The goal is clarity and craft — documents that look intentional without looking decorative. Callouts use background-tint + left border (3px). Code blocks use a dark surface (#1c1917) against the warm background.

Impeccable Integration

After Phase 2 writes the HTML file, if the impeccable skill is available and PRODUCT.md exists in the project, floreo invokes /impeccable polish [file] for a visual quality pass. Impeccable brings: OKLCH-aware color strategy, typography hierarchy enforcement (≥1.25 scale ratio between steps), deliberate spacing rhythm (not uniform padding), and the full impeccable design laws. After impeccable runs, floreo verifies the file remains self-contained — no external CSS or JS introduced.

The integration is conditional by design. Impeccable requires PRODUCT.md for project context. Without it, floreo skips Phase 3 and nudges: "Run /impeccable teach to enable automatic visual polish."

Enabling Phase 3

  1. 1
    Run /impeccable teach in this project directory — provide this document as context for the session
  2. 2
    Describe floreo's register as "document / editorial" — not product UI or marketing
  3. 3
    Describe design identity: warm stone neutrals, editorial serif headings (Georgia), system-ui body, accent color varies by document type, dark mode via CSS media query
  4. 4
    Describe anti-references: no decorative animations, no glassmorphism, no hero metrics, no gradient text — clarity over decoration. Documents exist to be read, not admired.
  5. 5
    After PRODUCT.md is created, Phase 3 activates automatically on the next floreo run
